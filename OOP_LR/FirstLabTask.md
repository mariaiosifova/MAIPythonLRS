
# Лабораторная работа: "Объектно-ориентированная реализация структур данных"

## Цель работы
Отработка тем по ООП (поля и методы класса, конструкторы, наследование, полиморфизм, перегрузка операторов), работа с исключениями, создание библиотек классов, реализация базовых структур данных.

## Теоретическая часть

### Структуры данных, рассматриваемые в работе:

1. **Односвязный список** - структура, где каждый элемент (узел) содержит данные и ссылку на следующий элемент
2. **Двусвязный список** - каждый узел содержит ссылки на следующий и предыдущий элементы
3. **Стек** - структура с принципом LIFO (Last In, First Out)
4. **Очередь** - структура с принципом FIFO (First In, First Out)

### Основные принципы ООП, применяемые в работе:

- **Инкапсуляция** - объединение данных и методов в классах
- **Наследование** - создание новых классов на основе существующих
- **Полиморфизм** - переопределение методов в наследниках
- **Перегрузка операторов** - определение поведения стандартных операторов

## Задание на лабораторную работу

### Часть 1. Базовый класс Node (Узел)

Создать класс `Node`, представляющий узел структуры данных:

**Поля класса:**
- `data` - данные, хранящиеся в узле
- `next` - ссылка на следующий узел (по умолчанию None)

**Методы:**
1. Конструктор `__init__` должен поддерживать три режима работы:
   - Без параметров: создает узел с `data = 0` и `next = None`
   - С одним параметром: `Node(5)` - создает узел с данными 5 и `next = None`
   - С двумя параметрами: `Node(5, next_node)` - создает узел с данными 5 и ссылкой на следующий узел

2. Метод `clear()` - обнуляет данные (устанавливает в 0) и убирает ссылку на следующий узел (устанавливает None)

3. Переопределить метод `__str__` для строкового представления узла (например, `"Node(5)"`)

### Часть 2. Односвязный список (SinglyLinkedList)

Создать класс `SinglyLinkedList`, который **использует композицию** с классом Node (содержит узлы, но не наследуется от Node).

**Поля класса:**
- `head` - ссылка на первый узел списка (None для пустого списка)
- `size` - количество элементов в списке (опционально)

**Методы добавления элементов:**
1. `prepend(data)` - добавление элемента в начало списка
2. `append(data)` - добавление элемента в конец списка
3. `insert_after(index, data)` - добавление элемента после элемента с указанным индексом
4. `insert_before(index, data)` - добавление элемента перед элементом с указанным индексом

**Методы удаления элементов:**
1. `remove_first()` - удаление первого элемента
2. `remove_last()` - удаление последнего элемента
3. `remove_at(index)` - удаление элемента по индексу
4. `remove_value(value)` - удаление первого вхождения указанного значения

**Методы поиска элементов:**
1. `find(value)` - поиск индекса первого вхождения значения (возвращает индекс или -1)
2. `find_all(value)` - поиск индексов всех вхождений значения (возвращает список индексов)
3. `contains(value)` - проверка наличия значения в списке (возвращает True/False)
4. `get(index)` - получение значения элемента по индексу

**Дополнительные методы:**
1. `__len__` - возвращает количество элементов
2. `__str__` - строковое представление списка (например, `"1 -> 2 -> 3 -> None"`)
3. `__iter__` - сделать список итерируемым
4. `clear()` - очистка списка

### Часть 3. Двусвязный список (DoublyLinkedList)

Создать класс `DoublyLinkedList`, используя **наследование** для узла:

Сначала создать класс `DoubleNode`, наследующий от Node:
```python
class DoubleNode(Node):
    def __init__(self, data=0, next=None, prev=None):
        super().__init__(data, next)
        self.prev = prev  # ссылка на предыдущий узел
```

Затем создать класс `DoublyLinkedList`.

**Поля класса:**
- `head` - ссылка на первый узел
- `tail` - ссылка на последний узел (опционально)

**Методы (аналогично односвязному списку, но с учетом двусвязности):**
- Добавление: `prepend`, `append`, `insert_after`, `insert_before`
- Удаление: `remove_first`, `remove_last`, `remove_at`, `remove_value`
- Поиск: `find`, `find_all`, `contains`, `get`
- Дополнительные: `__len__`, `__str__`, `__iter__`, `clear`

**Особенности для двусвязного списка:**
- При добавлении/удалении нужно обновлять обе ссылки (next и prev)
- Можно реализовать проход по списку как с начала, так и с конца

### Часть 6. Система исключений

Создать иерархию классов для обработки ошибок, специфичных для структур данных:

```python
class StructureError(Exception):
    """Базовый класс для всех исключений структур данных"""
    pass

class EmptyStructureError(StructureError):
    """Ошибка при попытке операции с пустой структурой"""
    def __init__(self, structure_name, message="Structure is empty"):
        self.structure_name = structure_name
        super().__init__(f"{message}: {structure_name}")

class IndexError(StructureError):
    """Ошибка при обращении по несуществующему индексу"""
    def __init__(self, index, message="Index out of range"):
        self.index = index
        super().__init__(f"{message}: {index}")

class ValueNotFoundError(StructureError):
    """Ошибка при поиске несуществующего значения"""
    def __init__(self, value, message="Value not found"):
        self.value = value
        super().__init__(f"{message}: {value}")
```
